# 🤝 智能上下文交接工作流 (Context Handoff)

**目的**：当当前对话（Session）过长，导致大模型遗忘上下文、出现幻觉或响应变慢时，强制收拢当前的状态，形成一份高度浓缩的交接文档，然后开启新对话。

## 阶段 1：状态快照收集 (Snapshot Collection)

Agent 必须主动收集以下信息：
1. **工作区状态**：执行 `git status` 和 `git diff` 读取当前未提交的代码变更。
2. **任务进度**：如果存在规划文档（如 `implementation_plan.md` 或 `task.md`）或近期的 `CHANGELOG.md`，读取它们的最后状态。
3. **遇到阻碍**：总结当前卡住的 Bug、报错信息或未决定的技术路线。

## 阶段 2：生成专属接力文档 (Generate Handoff Doc)

Agent 会在 `.agents/` 目录中创建或覆盖 `handoff.md`，文档必须是**写给接手的新 AI 审阅的**（以 Prompt/Context 投喂格式为主），结构如下：

```markdown
# 🤖 Agent Context Handoff (给新接手 AI 的上下文)

> **TO THE NEW ASSISTANT:** 你接手了一个正在进行中的任务。请仔细阅读以下上下文，继承前置规则与进度，然后无缝继续。

## 🎯 当前终极目标 (Mission)
// 简短说明这个项目在做什么，用户现在的根本诉求是什么。

## ✅ 已完成节点 (Accomplished)
// 列出最近完成的核心功能、修复的 Bug，只挑最重要的说，帮助新 AI 建立成就基线。

## 🚧 正在进行与中断点 (WIP & Interrupted At)
// 详细说明你（前任 AI）当前卡在哪个文件的哪一行，或者刚刚完成了哪个步骤准备做下一步。未提交的代码状态是什么。

## ⚠️ 避坑指南与硬性规矩 (Quirks & Rules)
// **极其重要**：如果上文对话中，用户发了脾气、或者遇到了某个底层库的离谱 Bug，或是约定了某个特有写法，必须在这里记下来！防止新 AI 重复踩同样的坑被用户骂。

## ⏭️ 你的第一步任务 (Next Steps for YOU)
// 明确指示新 AI: “阅读这段话后，请立即去检查某某文件”，或者“请立刻回复用户你已准备好继续修复某某 Bug”。
```

## 阶段 3：交接仪式 (Handoff Ceremony)

生成 `handoff.md` 后，Agent 必须向用户输出以下结束语：

> 🎉 **上下文接力棒已备好！**
> 
> 当前的对话历史已经变得很长。为了让我（或者新的我）保持清醒的认知和极速响应，**请现在关闭当前对话，新开一个 Chat/Composer 窗口**。
> 
> 在新对话里，你可以直接对我说：
> `“请读取 .agents/handoff.md，继承上下文记忆，然后继续工作。”`
