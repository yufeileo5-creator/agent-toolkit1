# 🤝 智能上下文交接工作流 (Context Handoff)

**目的**：当当前对话（Session）过长，导致大模型遗忘上下文、出现幻觉或响应变慢时，强制收拢当前的状态，形成一份高度浓缩的交接文档，然后开启新对话。

> [!IMPORTANT]
> 本工作流的核心目标不仅是传递「进度快照」，更是传递「推理链」——确保新 AI 知道**为什么这样做、否决了什么、约定了什么**，从而杜绝方向偏离。

## 阶段 1：深度上下文与状态审计 (Deep Context & State Audit)

Agent 必须主动收集以下信息：

### 1.1 工作区状态
- 执行 `git status` 和 `git diff --stat` 读取当前未提交的代码变更。
- 如果存在 `.dsp/graph.json`，检查 DSP 图谱是否与代码最新状态对齐，是否存在待消解的循环依赖。

### 1.2 任务进度
- 读取当前对话 artifact 中的 `task.md` 和 `implementation_plan.md`（如果存在）。
- 读取项目级 `CHANGELOG.md` 最近的条目。

### 1.2.1 Implementation Plan 沉淀（强制）

> [!IMPORTANT]
> 本步骤是防止 Plan 知识断裂的**兜底安全网**。Plan 的主沉淀时机在 `sdd:plan` 技能（Plan 审批通过后立刻同步），但用户可能未使用 `/plan` 工作流或中途跳过了同步。此步骤确保在 handoff 时做最后检查和补漏。

如果当前对话 artifact 中存在 `implementation_plan.md`：

1. **评估 Plan 状态**：判断 Plan 是「已完成」「进行中」还是「已废弃」
2. **同步到 PLANS.md**：
   - **已完成** → 移入 `PLANS.md`「历史计划归档」，保留核心决策和否决方案（精华不超过 30 行）
   - **进行中** → 更新 `PLANS.md`「当前活跃计划」，包含精华摘要 + 未完成步骤
   - **已废弃** → 移入「历史计划归档」并标注废弃原因
3. **Handoff 内嵌摘要**：在交接文档「🔥 当前正在进行的工作」章节写入 `### 实施计划摘要`（见阶段 2 模板）
4. **清理旧引用**：`PLANS.md` 中不得出现「参见对话 artifact xxx」的悬空引用，必须将内容内联或标注为「已归档 → _versions/」

**即使对话 artifact 中不存在 `implementation_plan.md`**（如快速开发未经正式规划流程），也必须执行：

5. **对齐检查**：读取 `PLANS.md` 的「当前活跃计划」，与本期对话实际完成的工作对比，判断是否存在漂移
   - 若 `PLANS.md` 的描述已过时 → 直接更新为本期工作的实际状态
   - 若本期工作完全未在 `PLANS.md` 中体现 → 补写对应条目（已完成 ✅ 或进行中 🟡）

### 1.3 质量纪律状态
- **强制**执行一次测试命令（`npx vitest run`、`npm test` 或项目约定的测试命令），准确记录通过率、失败用例和 Warning。
- 如有 lint 工具，执行 `npm run lint` 并记录输出。
- 记录结果到交接文档的「质量与架构防线」段落中。

### 1.4 决策考古 (Decision Archaeology)

> [!CAUTION]
> 这是本工作流最关键、也是与旧版最大的差异。AI 必须**回溯整个对话历史**，主动提取以下内容：

1. **关键设计决策**: 对话中讨论并达成共识的设计选择，包括：
   - 选了哪个方案
   - 还考虑过哪些备选方案
   - 每个备选方案被否决的**具体原因**
2. **明确否决的方案**: 用户或 AI 在讨论中提出但最终被放弃的技术路线，必须逐条列出，**附带否决原因**。
3. **用户偏好与隐式约定**: 用户在对话中表达的偏好（如"不要用 X 库"、"这个模块必须保持纯函数"），即使没有写入任何文档，也必须收集。
4. **遇到的坑和 workaround**: 开发过程中遇到的底层库 Bug、浏览器兼容性问题、特殊 workaround，防止新 AI 重复踩坑。

### 1.5 ADR 自动沉淀

如果决策考古中发现了**跨期有效的架构决策**（不仅仅是本期的实现细节，而是会影响后续多期开发的设计原则），Agent 必须：
1. 在 `docs/adr/` 目录下创建对应的 ADR 文件，使用递增编号（如 `0003-xxx.md`）。
2. ADR 格式遵循项目已有的 ADR 模板（参考 `docs/adr/` 中的现有文件）。
3. 在交接文档中引用新创建的 ADR。

## 阶段 3：交接仪式 (Handoff Ceremony)

生成交接文档后，Agent 必须向用户输出以下结束语：

> 🎉 **上下文接力棒已备好！**
>
> 当前的对话历史已经变得很长。为了让我（或者新的我）保持清醒的认知和极速响应，**请现在关闭当前对话，新开一个 Chat/Composer 窗口**。
>
> 在新对话里，你可以直接对我说：
> `"请读取 docs/handoff.md 和 docs/adr/ 下的所有 ADR 文件，继承上下文记忆，然后继续工作。"`
>
> ⚠️ **对齐校验提示**：为了确保新 AI 没有偏离你之前讨论好的方向，建议你在新窗口里先要求 AI：
> `"先复述你对当前设计方案、关键约束和否决方案的理解，我确认后你再动手。"`
