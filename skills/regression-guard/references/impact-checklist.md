# 回归影响检查清单

本清单用于 `regression-guard` 技能的 Step 3，按变更类型分类检查回归风险。

## 通用检查项

对每个受影响的模块，逐项确认：

### 接口兼容性

- [ ] 导入语句是否仍然解析正确（无 "Module not found"）
- [ ] 使用的导出名称是否仍然存在（无 "is not exported"）
- [ ] 函数参数数量和类型是否匹配
- [ ] 返回值结构是否与消费方的解构一致
- [ ] 泛型约束是否仍然满足

### 行为一致性

- [ ] 函数的副作用是否保持不变（如：是否仍然触发事件、更新状态）
- [ ] 异步行为是否保持不变（如：Promise 的 resolve/reject 时机）
- [ ] 错误处理路径是否保持不变（如：抛出的异常类型）
- [ ] 边界值处理是否保持不变（如：空数组、null、undefined）

### 状态管理

- [ ] Store/Context 的 shape 是否兼容
- [ ] 选择器（selector）是否仍然能正确读取数据
- [ ] 状态更新事件是否仍然正确触发

### 样式与布局

- [ ] CSS class 名称是否保持不变
- [ ] 组件的 DOM 结构是否影响父级布局
- [ ] 响应式断点是否仍然正确

---

## 项目特定检查项

### 排版引擎相关

针对本项目的 AI 布局引擎，额外检查：

- [ ] AI 生成的布局 JSON 结构是否与编辑器解析器兼容
- [ ] 图片块数量是否与文档中的图片数量一致
- [ ] 文本内容是否正确填充到矩形块中
- [ ] 坐标系（x, y, width, height）的单位和范围是否一致
- [ ] 渲染后的图片是否可见（非空白/非 0 尺寸）

### 文件处理相关

- [ ] 文件上传/解析流程是否正常
- [ ] 文件格式兼容性是否保持（docx, pptx, pdf）
- [ ] 提取的文本/图片是否完整

### API 调用相关

- [ ] API 端点路径是否正确
- [ ] 请求/响应的数据格式是否匹配
- [ ] 错误码处理是否覆盖
- [ ] 超时和重试机制是否正常
